579. Find Cumulative Salary of an Employee
Solved
Hard
Topics
Companies
Hint
SQL Schema
Pandas Schema

Table: Employee

+-------------+------+
| Column Name | Type |
+-------------+------+
| id          | int  |
| month       | int  |
| salary      | int  |
+-------------+------+
(id, month) is the primary key (combination of columns with unique values) for this table.
Each row in the table indicates the salary of an employee in one month during the year 2020.

 

Write a solution to calculate the cumulative salary summary for every employee in a single unified table.

The cumulative salary summary for an employee can be calculated as follows:

  • For each month that the employee worked, sum up the salaries in that month and the previous two months. This is their 3-month sum for that month. If an employee did not work for the company in previous months, their effective salary for those months is 0.
  • Do not include the 3-month sum for the most recent month that the employee worked for in the summary.
  • Do not include the 3-month sum for any month the employee did not work.

Return the result table ordered by id in ascending order. In case of a tie, order it by month in descending order.

The result format is in the following example.

 

Example 1:

Input: 
Employee table:
+----+-------+--------+
| id | month | salary |
+----+-------+--------+
| 1  | 1     | 20     |
| 2  | 1     | 20     |
| 1  | 2     | 30     |
| 2  | 2     | 30     |
| 3  | 2     | 40     |
| 1  | 3     | 40     |
| 3  | 3     | 60     |
| 1  | 4     | 60     |
| 3  | 4     | 70     |
| 1  | 7     | 90     |
| 1  | 8     | 90     |
+----+-------+--------+
Output: 
+----+-------+--------+
| id | month | Salary |
+----+-------+--------+
| 1  | 7     | 90     |
| 1  | 4     | 130    |
| 1  | 3     | 90     |
| 1  | 2     | 50     |
| 1  | 1     | 20     |
| 2  | 1     | 20     |
| 3  | 3     | 100    |
| 3  | 2     | 40     |
+----+-------+--------+
Explanation: 
Employee '1' has five salary records excluding their most recent month '8':
- 90 for month '7'.
- 60 for month '4'.
- 40 for month '3'.
- 30 for month '2'.
- 20 for month '1'.
So the cumulative salary summary for this employee is:
+----+-------+--------+
| id | month | salary |
+----+-------+--------+
| 1  | 7     | 90     |  (90 + 0 + 0)
| 1  | 4     | 130    |  (60 + 40 + 30)
| 1  | 3     | 90     |  (40 + 30 + 20)
| 1  | 2     | 50     |  (30 + 20 + 0)
| 1  | 1     | 20     |  (20 + 0 + 0)
+----+-------+--------+
Note that the 3-month sum for month '7' is 90 because they did not work during month '6' or month '5'.

Employee '2' only has one salary record (month '1') excluding their most recent month '2'.
+----+-------+--------+
| id | month | salary |
+----+-------+--------+
| 2  | 1     | 20     |  (20 + 0 + 0)
+----+-------+--------+

Employee '3' has two salary records excluding their most recent month '4':
- 60 for month '3'.
- 40 for month '2'.
So the cumulative salary summary for this employee is:
+----+-------+--------+
| id | month | salary |
+----+-------+--------+
| 3  | 3     | 100    |  (60 + 40 + 0)
| 3  | 2     | 40     |  (40 + 0 + 0)
+----+-------+--------+

WITH cte AS(
SELECT id, month, Salary, ROW_NUMBER() OVER (PARTITION BY Id ORDER BY Month DESC) AS n
FROM Employee)
SELECT E1.Id, E1.Month, E1.Salary + IFNULL(E2.Salary, 0) + IFNULL(E3.Salary, 0) AS Salary
FROM cte AS E1
LEFT JOIN cte AS E2 ON E1.id = E2.id AND E1.Month - 1 = E2.Month
LEFT JOIN cte AS E3 ON E2.id = E3.id AND E2.Month - 1 = E3.Month
WHERE E1.n <> 1;

Approach

  1. The first recursive CTE creates a single-column table called 'months' with 1-12 values;
  2. 'empmonths' generates all combinations of (id, month)
  3. 'whole' is the 'empmonths' with 'salary' pasted on the right, if no salary then null values
  4. 'selectedwhole' calculates the sum(salary) for every month (including the ones without working), seeing nulls as 0. This step guarantees that missing months are also included.
  5. finally, left join 'selectedwhole' to 'employee' excludes months that employees don't work, and the WHERE clause excludes the most recent month.

Code

with recursive months as
(
	select 1 as 'month'
	union all
	select month+1 from months
	where month < 12
),
empmonths as (
    select * from
    (select distinct id from employee) sub1
    cross join
    (select * from months) sub2
),
whole as (
    select *
    from empmonths
    left join employee
    using (id, month)
),
selectedwhole as (
    select id, month,
        sum(ifnull(salary, 0)) over (partition by id order by month
                                    range between 2 preceding and current row)
        as Salary
    from whole
)
select e.id, e.month, s.Salary
from employee e
left join selectedwhole s
using (id, month)
where (e.id, e.month) not in (
    select id, max(month)
    from employee
    group by id
)
order by id, month desc
